## WasmMend

WebAssembly (Wasm) promises seamless reuse of C/C++ codebases as portable, fast, sandboxed binaries. In practice, however, this promise often breaks: recent studies show that cross-compiling the same C/C++ source to Wasm and to a native binary frequently leads to silent runtime divergence or crashes, owing to library implementation differences or compiler bugs. Since the root causes live in platform-level runtime behavior and are invisible in the source code, even state-of-the-art LLM-based repair agents usually fail to fix these discrepancies. 

In this paper, we present WasmMend, the first system to automatically repair Wasm-native functional discrepancies. WasmMend converts an unbounded source-level search into a focused reasoning task: it first conducts a novel differential trace analysis to localize the function at which Wasm and native executions first diverge. Guided by this localization, WasmMend then invokes LLM agents to reason about root causes and generate patches that eliminate the divergent behavior. Experiments on real-world C/C++ projects show that WasmMend achieves a patch rate of 70.0\%, compared to 54.5\% for the source-only agentic baseline and 50.2\% for a baseline augmented with repair-time LLM instrumentation, demonstrating the value of divergence-guided reasoning for cross-platform repair.
